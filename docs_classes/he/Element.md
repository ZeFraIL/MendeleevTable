# תיאור המחלקה Element

## 1. מידע כללי
*   **שם המחלקה:** `Element`
*   **סוג:** Normal Class (POJO) — אובייקט נתונים פשוט.
*   **מטרה:** מתארת יסוד כימי בודד מהטבלה המחזורית. זוהי "תבנית" לשמירת נתונים: שם היסוד, מספר אטומי, מסה וכתובת התמונה.
*   **אינטראקציה:** נוצרת ב-`MainActivity`, נשמרת ברשימה ומועברת ל-`ElementDetailActivity`.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `symbol` | `String` | סמל כימי (למשל "H", "O"). | תצוגה ברשימה ובפרטים |
| `name` | `String` | שם מלא (למשל "Hydrogen"). | תצוגה ברשימה ובפרטים |
| `atomicNumber` | `int` | מספר אטומי. | מיון ותצוגה |
| `atomicMass` | `double` | מסה אטומית. | תיאור מפורט |
| `imageUrl` | `String` | כתובת התמונה באינטרנט. | טעינת תמונה במתאם |

## 6. אינטראקציה עם רכיבים אחרים
*   **Serializable:** המחלקה מממשת את ממשק `Serializable`. זה מאפשר להפוך את האובייקט לרצף בתים כדי להעביר אותו בין מסכים דרך `Intent`.

## 8. הסבר במילים פשוטות
`Element` הוא כמו **טופס** או **פרופיל**. יש בו שורות כמו "שם", "סמל", "משקל". הטופס עצמו הוא המחלקה. טופס מלא עבור יסוד ספציפי (למשל מימן) הוא **אובייקט** של המחלקה. האפליקציה יוצרת 118 טפסים כאלה ומציגה לנו אותם.

---
⚠️ **הערת באג:** ב-`ElementDetailActivity` יש ניסיון לקבל נתונים דרך `getParcelableExtra`, אך המחלקה מממשת `Serializable`. יש לתקן זאת ל-`getSerializableExtra`.
