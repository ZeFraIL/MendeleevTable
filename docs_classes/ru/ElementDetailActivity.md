# Описание класса ElementDetailActivity

## 1. Общая информация
*   **Имя класса:** `ElementDetailActivity`
*   **Тип:** Activity (Активность) — экран деталей.
*   **Назначение:** Отображает подробную информацию о конкретном выбранном химическом элементе: полное название, символ, номер и большую картинку.
*   **Взаимодействие:** Получает данные от `MainActivity` через объект `Intent`.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Информация о среде приложения. | Вспомогательные цели |
| `executorService` | `ExecutorService` | Группа из 4 потоков для быстрой загрузки изображения. | `loadImageWithExecutor` |

## 3. Методы класса

### Метод: `onCreate`
*   **Тип:** `protected`
*   **Что делает:** 
    1.  Извлекает объект `Element` из "посылки" (`Intent`).
    2.  Связывает переменные с визуальными элементами экрана.
    3.  Заполняет текстовые поля данными.
    4.  Вызывает метод для загрузки картинки.
*   **Когда вызывается:** При открытии экрана (клике на элемент в списке).

### Метод: `loadImageWithExecutor`
*   **Тип:** `private`
*   **Что делает:** Запускает фоновую задачу для скачивания картинки по URL. После завершения скачивания использует `imageView.post()`, чтобы вернуться в главный поток и показать картинку пользователю.
*   **Зачем:** Картинки могут быть тяжелыми, их нельзя качать в главном потоке.

## 4. Жизненный цикл
*   **`onCreate()`**: Инициализация данных и экрана.
*   **`onDestroy()`**: Важный этап — закрытие `executorService`. Если этого не сделать, потоки продолжат работать в памяти даже после закрытия окна, что приведет к "утечке" памяти.

## 5. Взаимодействие с интерфейсом (UI)
*   Макет: `activity_element_detail.xml`.
*   Элементы: `tvName`, `tvSymbol`, `tvAtomicNumber`, `tvAtomicMass`, `ivElement`.

## 6. Взаимодействие с другими компонентами
*   Получает объект через `Intent`. 
*   **Внимание!** В коде используется `getParcelableExtra`, но объект `Element` передается как `Serializable`. Чтобы это работало без ошибок, в `MainActivity` передача должна быть корректной, либо нужно использовать `getSerializableExtra`.

## 7. Общая логика
1. Экран открывается.
2. Проверяет: прислали ли нам данные? Если нет — закрывается с ошибкой.
3. Если данные есть, рисует их на экране.
4. Отправляет "запрос" в интернет на получение картинки.

## 8. Объяснение простыми словами
`ElementDetailActivity` — это **увеличительное стекло**. На главном экране мы видим много элементов вкратце. Когда мы нажимаем на один из них, это "стекло" открывается на весь экран и показывает нам всё самое интересное про один конкретный элемент.

---
⚠️ **Совет по улучшению:** Вместо ручного создания `ExecutorService` для каждой картинки, лучше использовать готовую библиотеку (Coil/Glide). Это сократит код с 30 строк до 1.
