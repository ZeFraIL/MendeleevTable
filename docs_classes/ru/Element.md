# Описание класса Element

## 1. Общая информация
*   **Имя класса:** `Element`
*   **Тип:** Normal Class (POJO) — простой объект с данными.
*   **Назначение:** Описывает один химический элемент из таблицы Менделеева. Это "чертеж" или "шаблон", по которому хранятся данные: как элемент называется, какой у него номер, масса и где взять его изображение.
*   **Взаимодействие:** Объекты этого класса создаются в `MainActivity`, хранятся в списке и передаются в `ElementDetailActivity` для отображения подробностей.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `symbol` | `String` | Химический символ (например, "H", "O"). | Отображение в списке и деталях |
| `name` | `String` | Полное название элемента (например, "Hydrogen"). | Отображение в списке и деталях |
| `atomicNumber` | `int` | Атомный номер (порядковый номер в таблице). | Сортировка и отображение |
| `atomicMass` | `double` | Атомная масса элемента. | Детальное описание |
| `imageUrl` | `String` | Ссылка (URL) на изображение элемента в интернете. | Загрузка картинки в адаптере |

## 3. Методы класса

### Метод: Конструктор `Element`
*   **Тип:** `public`
*   **Что делает:** Заполняет все поля объекта данными при его создании. Без этого метода поля были бы пустыми.
*   **Параметры:** Принимает все 5 параметров (символ, имя, номер, массу, ссылку).

### Методы: `getSymbol()`, `getName()` и др. (Геттеры)
*   **Тип:** `public`
*   **Возвращаемое значение:** Значение соответствующего поля.
*   **Зачем нужны:** Поля помечены как `private` (инкапсуляция). Это значит, что другие классы не могут "влезть" в данные напрямую, а должны вежливо попросить их через эти методы.

## 4. Жизненный цикл
У обычного класса нет жизненного цикла Android. Он "живет", пока на него есть ссылка в памяти (например, пока он находится в списке `elementList`).

## 6. Взаимодействие с другими компонентами
*   **Serializable:** Класс реализует интерфейс `Serializable`. Это "магия", которая позволяет превратить объект в набор байтов, чтобы передать его между экранами (Activity) через `Intent`.

## 7. Общая логика
Это просто контейнер. Если нам нужно передать информацию о "Золоте", мы создаем объект `new Element("Au", "Gold", 79, ...)`. Теперь этот объект "Золото" можно передавать по всему приложению как одну посылку.

## 8. Объяснение простыми словами
`Element` — это как **анкета**. В ней есть строчки: "Имя", "Фамилия", "Возраст". Сама анкета — это класс. А заполненная анкета на конкретного человека (например, на Водород) — это **объект** класса. Приложение создает 118 таких анкет и показывает их нам.

---
⚠️ **Важное замечание:** В классе `ElementDetailActivity` данные пытаются получить через `getParcelableExtra`. Однако класс `Element` реализует `Serializable`. Это **ошибка** в текущем коде: нужно либо использовать `getSerializableExtra`, либо изменить класс `Element`, чтобы он реализовывал `Parcelable`.
