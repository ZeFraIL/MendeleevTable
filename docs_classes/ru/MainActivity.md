# Описание класса MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность) — основное окно приложения.
*   **Назначение:** Является "входной точкой" и главным экраном приложения. Класс отвечает за загрузку списка химических элементов из интернета, его обработку и отображение пользователю в виде списка.
*   **Взаимодействие:** 
    *   Использует `ElementAdapter` для отображения данных.
    *   Создает список объектов `Element`.
    *   Запускает `ElementDetailActivity` при выборе элемента.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `recyclerView` | `RecyclerView` | Список-контейнер для отображения элементов на экране. | `onCreate` |
| `adapter` | `ElementAdapter` | "Мост" между данными и списком. Управляет созданием строк списка. | `onCreate`, `parseJSON` |
| `elementList` | `ArrayList<Element>` | Динамический массив, в котором хранятся все загруженные элементы. | Весь класс |
| `executorService` | `ExecutorService` | Служба для выполнения задач в фоновом потоке (чтобы приложение не "зависало"). | `loadElementsFromJSON`, `onDestroy` |

## 3. Методы класса

### Метод: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (состояние приложения).
*   **Что делает:** Выполняет начальную настройку: устанавливает макет (layout), находит список `RecyclerView` в разметке, инициализирует адаптер и запускает процесс загрузки данных.
*   **Когда вызывается:** Система вызывает его один раз при создании окна активности.
*   **Важно:** Здесь нельзя выполнять долгие операции (например, загрузку из сети), иначе экран будет белым.

### Метод: `loadElementsFromJSON`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** 
    1.  Создает "задание" для фонового потока.
    2.  Подключается к GitHub по URL.
    3.  Скачивает JSON-текст (специальный формат данных).
    4.  Передает текст в метод `parseJSON`.
*   **Когда вызывается:** Из `onCreate` сразу после запуска.
*   **Важно:** Сетевые запросы в Android ОБЯЗАТЕЛЬНО должны быть в фоновом потоке (используем `executorService`).

### Метод: `parseJSON`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `String json` (текст в формате JSON).
*   **Что делает:** Проходит по тексту, вытаскивает название, символ, массу и ссылку на картинку для каждого элемента, создает объект `Element` и добавляет его в список. В конце приказывает адаптеру обновить экран.
*   **Когда вызывается:** После успешной загрузки данных.

## 4. Жизненный цикл
*   **`onCreate()`**: Создание окна, настройка списка.
*   **`onDestroy()`**: Вызывается при закрытии приложения. Здесь мы останавливаем `executorService`, чтобы он не потреблял память впустую.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** Используется `RecyclerView` (динамический список).
*   **Связь:** Поиск элемента через `findViewById(R.id.recyclerView)`.

## 6. Взаимодействие с другими компонентами
*   Данные загружаются через **API (GitHub)**.
*   Для передачи данных на другой экран обычно используется `Intent` (в коде `MainActivity` переход на детали не реализован в явном виде, но подразумевается через адаптер).

## 7. Общая логика
1. Система создает окно (`onCreate`).
2. Приложение идет в интернет (`loadElementsFromJSON`).
3. Получает текст, превращает его в список объектов (`parseJSON`).
4. Список отображается на экране через адаптер.

## 8. Объяснение простыми словами
Представьте, что `MainActivity` — это **менеджер ресторана**. 
1. Он готовит зал (устанавливает `RecyclerView`). 
2. Затем он звонит на склад (в интернет) и заказывает продукты (`loadElementsFromJSON`). 
3. Когда продукты привозят, он проверяет накладную и раскладывает их по коробкам (`parseJSON`). 
4. В конце он отдает продукты повару (`Adapter`), чтобы тот красиво разложил их по тарелкам для гостей.

---
⚠️ **Совет по коду:** В методе `parseJSON` используется `runOnUiThread`. Это критически важно, так как обновлять экран можно только из главного потока, а данные мы обрабатывали в фоновом.
