# وصف فئة MainActivity

## 1. معلومات عامة
*   **اسم الفئة:** `MainActivity`
*   **النوع:** Activity (نشاط) — النافذة الرئيسية للتطبيق.
*   **الغرض:** هي "نقطة الدخول" والشاشة الرئيسية. هذه الفئة مسؤولة عن تحميل قائمة العناصر الكيميائية من الإنترنت، ومعالجتها، وعرضها للمستخدم في شكل قائمة.
*   **التفاعل:** 
    *   تستخدم `ElementAdapter` لعرض البيانات.
    *   تنشئ قائمة من كائنات `Element`.
    *   تفتح `ElementDetailActivity` عند اختيار عنصر.

## 2. المتغيرات
| الاسم | النوع | الغرض | أين يستخدم |
| :--- | :--- | :--- | :--- |
| `recyclerView` | `RecyclerView` | حاوية القائمة لعرض العناصر على الشاشة. | `onCreate` |
| `adapter` | `ElementAdapter` | "جسر" بين البيانات والقائمة. | `onCreate`, `parseJSON` |
| `elementList` | `ArrayList<Element>` | مصفوفة ديناميكية تخزن جميع العناصر المحملة. | كامل الفئة |
| `executorService` | `ExecutorService` | خدمة لتشغيل المهام في خلفية النظام (حتى لا "يتجمد" التطبيق). | `loadElementsFromJSON`, `onDestroy` |

## 3. الأساليب (Methods)

### الأسلوب: `onCreate`
*   **الغرض:** القيام بالإعدادات الأولية: تحديد التنسيق (Layout)، وتهيئة المحول (Adapter)، وبدء عملية تحميل البيانات.
*   **متى يستدعى:** يستدعيه النظام مرة واحدة عند إنشاء نافذة النشاط.

### الأسلوب: `loadElementsFromJSON`
*   **الغرض:** 
    1. إنشاء "مهمة" لخيط الخلفية.
    2. الاتصال بـ GitHub عبر رابط URL.
    3. تحميل نص JSON.
    4. تمرير النص إلى أسلوب `parseJSON`.

## 8. شرح بكلمات بسيطة
تخيل أن `MainActivity` هي **مدير مطعم**.
1. يقوم بتجهيز قاعة الطعام (إعداد الـ `RecyclerView`).
2. يتصل بالمخزن (الإنترنت) ويطلب المكونات (`loadElementsFromJSON`).
3. عندما تصل المكونات، يفحص الفاتورة ويضعها في صناديق (`parseJSON`).
4. أخيراً، يعطي المكونات لـ **الطاهي** (`Adapter`) ليقوم بترتيبها بشكل جميل للضيوف.
