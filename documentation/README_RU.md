🎯 Цель проекта
•	Какую задачу решает приложение
Приложение предоставляет удобный доступ к периодической системе химических элементов Менделеева, позволяя просматривать список элементов с их основными характеристиками и изображениями.

•	Почему эта задача важна
Периодическая таблица является фундаментальным инструментом для студентов, ученых и всех, кто интересуется химией. Мобильное приложение делает эти данные доступными в любом месте и в любое время.

•	Целевая аудитория
Студенты, преподаватели химии, научные сотрудники и школьники.

📌 Требования к приложению
Функциональные требования
•	Загрузка актуальных данных об элементах из удаленного JSON-ресурса.
•	Отображение списка элементов в виде карточек (RecyclerView).
•	Отображение подробной информации о выбранном элементе в отдельном окне.
•	Загрузка и отображение изображений элементов.

Нефункциональные требования
•	Производительность: Быстрая загрузка и плавный скроллинг списка.
•	Удобство использования: Интуитивно понятный интерфейс в стиле Material Design.
•	Надёжность: Обработка ошибок сетевого соединения и корректный парсинг данных.

🧠 Общая архитектура
•	Выбранный подход:
–	MVC (Model-View-Controller)

•	Почему выбран именно он
Для данного проекта (просмотр списка и деталей) архитектура MVC является оптимальной, так как она проста в реализации и не перегружает проект лишними абстракциями при небольшом количестве логики.

•	Основные компоненты системы
–	Model: Класс `Element`, представляющий сущность химического элемента.
–	View: XML-разметки (`activity_main.xml`, `item_element.xml`, `activity_element_detail.xml`).
–	Controller: `MainActivity` и `ElementDetailActivity`, управляющие логикой и взаимодействием с пользователем.

Для дальнейшего масштабирования рекомендуется разделить код на пакеты `data`, `ui`, `model`, что позволит легко добавлять новые функции, такие как локальная база данных (Room) или поиск.

🧩 Подробное описание классов
📌 Класс: MainActivity
Роль:
Главный экран приложения.
Ответственность:
Загрузка данных из сети, инициализация RecyclerView, обработка переходов на экран деталей.
Основные методы: 
- onCreate() — инициализация интерфейса и запуск загрузки данных.
- loadElementsFromJSON() — выполнение сетевого запроса через ExecutorService.
- parseJSON() — обработка полученной строки и создание объектов Element.
Взаимодействие с другими классами:
Использует `ElementAdapter` для отображения списка и передает данные в `ElementDetailActivity`.
________________________________________
📌 Класс: ElementDetailActivity
Роль:
Экран детальной информации.
Почему используется:
Для детального ознакомления с характеристиками конкретного химического элемента.
________________________________________
📌 Класс: ElementAdapter
Роль:
Связующее звено между данными (ArrayList<Element>) и графическим представлением (RecyclerView).

🔄 Диаграмма работы приложения
Опишите сценарий:
1. Пользователь запускает приложение.
2. `MainActivity` в фоновом потоке запрашивает JSON-файл с GitHub.
3. После получения ответа данные парсятся в список объектов `Element`.
4. `ElementAdapter` обновляет `RecyclerView` на главном экране.
5. При нажатии на элемент открывается `ElementDetailActivity`, куда передаются данные выбранного элемента.

🎨 UI/UX анализ
•	Почему интерфейс сделан именно так
Выбран карточный дизайн для списка, чтобы четко разделить информацию о каждом элементе. Использование Material Design обеспечивает привычный пользовательский опыт.

•	Какие принципы использованы:
–	Простота: Нет лишних элементов, фокус на данных.
–	Логичность: Переход от общего списка к деталям.
–	Доступность: Четкие шрифты и контрастные изображения.

•	Что можно улучшить
Добавить поиск по названию/символу, фильтрацию по группам и темную тему.

⚙️ Работа с потоками
Опишите: - Используются ли: 
- ExecutorService (в MainActivity и ElementDetailActivity)
- AsyncTask (в ElementAdapter для загрузки картинок)
•	Почему выбран этот способ
ExecutorService обеспечивает эффективное выполнение сетевых операций вне основного (UI) потока, предотвращая зависания.

•	Как предотвращаются:
–	зависания (ANR): Все сетевые запросы и парсинг выполняются в фоновых потоках.
–	утечки памяти: Использование shutdown() для ExecutorService в методе onDestroy().

💾 Работа с данными
•	Где хранятся данные
Данные загружаются динамически из удаленного JSON-файла.

•	Почему выбран этот способ
Это позволяет всегда иметь актуальный список элементов без необходимости обновлять само приложение.

•	Как обеспечивается:
–	сохранность: Данные хранятся в облаке (GitHub).
–	корректность: Использование блока try-catch при парсинге JSON.
